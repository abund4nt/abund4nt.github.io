Modular Exponentiation

La exponenciacion modular es un tipo de exponenciacion realizada sobre un modulo (\(mod\)), en esta division se calcula el residuo cuando un numero entero positivo \(b\) se eleva a \(e\) (\(b^{e}\)), luego esto es dividido por un entero positivo \(m\).

Ejemplo.

\(b = 5, e = 3 \text{ y } m = 13\)

La solucion es 8 ya que es el resto de dividir \(5^{3}\) por 13.

En el desafio nos piden.

To grab the flag, find the solution to 101^17 mod 22663

Para resolver el desafio podemos utilizar Python.

$ python3 -q
>>> pow(101, 17, 22663)
19906

Esta exponenciacion modular es utilizada para crear una funcion trampa.

Una funcion trampa es una funcion matematica cuyo calculo directo es facil, pero el calculo de la funcion inversa es muy complejo y requiere mucho poder de computo, como por ejemplo encontrar \(p\) y \(q\) con \(n\) de 100 cifras un ordenador tardaria mas de \(10^{36}\) años.

Public Keys

Nos explican que el modulo \(n\) es el producto de dos primos \(n = p \cdot q\) y que el valor del exponente publico \(e\) normalmente es \(65537\) o su equivalente 0x10001 en hexadecimal.

Para resolver el desafio debemos “encriptar” el numero 12 utilizando el exponente \(e = 65537\) y los numeros primos \(p = 17\) y \(q = 24\), utilizaremos Python para esto.

$ python3 -q
>>> e = 65537
>>> p = 17
>>> q = 23
>>> m = 12
>>> pow(m, e, p * q)
301

Euler’s Totient \(\phi\)

La funcion de Euler es una funcion importante en teoria de numeros. Si \(n\) es un numero entero positivo, entonces \(\phi(n)\) se define como la cantidad de enteros positivos menores a \(n\) y coprimos (es decir que su maximo comun divisor es igual a 1) con n.

Para calcular \(\phi\) en RSA se utiliza la siguiente formula.

$$ \phi = (p - 1) \cdot (q - 1) $$

En el desafio nos piden calcular \(\phi\) con los siguientes valores.

p = 857504083339712752489993810777
q = 1029224947942998075080348647219

Utilizaremos Python para calcular \(\phi\) y resolver el desafio.

$ python3 -q
>>> p = 857504083339712752489993810777
>>> q = 1029224947942998075080348647219
>>> phi = (p - 1) * (q - 1)
>>> phi
882564595536224140639625987657529300394956519977044270821168

Private Keys

La clave privada \(d\) en RSA se utiliza para desencriptar textos cifrados que fueron creados con su correspondiente clave publica, esta clave es la informacion secreta que nos permite invertir rapidamente la funcion de cifrado.

La clave \(d\) es el inverso multiplicativo modular del exponente \(e\) modulo \(\phi\).

En el desafio nos piden calcular \(d\) con los siguientes datos.

p = 857504083339712752489993810777
q = 1029224947942998075080348647219 
e = 65537

Utilizaremos Python para resolver el desafio.

python3 -q
>>> p = 857504083339712752489993810777
>>> q = 1029224947942998075080348647219
>>> e = 65537
>>> phi = (p - 1) * (q - 1)
>>> d = pow(e, -1, phi)
>>> d
121832886702415731577073962957377780195510499965398469843281

Firmar un mensajes se usa para asegurarte de que la persona que recibe el mensaje sepa que lo escribiste tu. Se siguen los siguientes pasos para encriptar el mensaje.

Encriptacion del mensaje

$$ c = m^{e^{0}} \text{ mod } n $$

Firma del mensaje

Para asegurar que el mensaje no sea alterado sin tu conocimiento, calculas un resumen criptografico del mensaje, conocido como hash \(H(m)\), luego se encripta este hash utilizando su clave privada \(s = H(m)^{d^{1}} \text{ mod } n^{1}\).

Con esto el destinatario puede verificar que el mensaje proviene realmente de ti de la siguiente manera.

  • Utiliza tu clave publica para desencriptar \(c\) y obtener el mensaje \(m = c^{d^{0}} \text{ mod } N^{0}\)

  • Luego utiliza la misma clave publica desencriptar \(S\) y obtener \(S = S^{e^{1}} \text{ mod }N^{1}\)

  • Calcula el hash del mensaje recibido \(H(m)\) y lo compara con \(S\). Si ambos son iguales, entonces el mensaje que recibio es exactamente el mensaje que firmaste.

Primes Part 1

Factoring

Nos explican que los primos pequeños no son buenos para RSA ya que pueden factorizarse con metodos modernos muy rapidamente, la idea es utilizar dos primos de almenos 1024 bits cada uno, así logramos obtener un modulo de 2048 bits, en RSA un modulo de 2048 bits se denomina RSA-2048.

Para resolver el desafio nos piden factorizar el siguiente numero de 150 510143758735509025530880200653196460532653147, utilizando el siguiente script en Python lo factorizamos.

from factordb.factordb import FactorDB

# Number
n = 510143758735509025530880200653196460532653147

# Factorize
f = FactorDB(n)
f.connect()
r = f.get_factor_list()

print(min(r))

Al ejecutar el script obtendremos el numero primo mas pequeño de ambos y resolvemos el desafio.

$ python3 solve.py
19704762736204164635843

Inferius Prime

Nos entregan dos archvivos output.txt y inferius.py. El .txt contine los parametros \(n\), \(e\) y \(ct\).

Y el archivo inferius.py es el codigo fuente.

#!/usr/bin/env python3

from Crypto.Util.number import getPrime, inverse, bytes_to_long, long_to_bytes, GCD

e = 3

# n will be 8 * (100 + 100) = 1600 bits strong which is pretty good
while True:
    p = getPrime(100)
    q = getPrime(100)
    phi = (p - 1) * (q - 1)
    d = inverse(e, phi)
    if d != -1 and GCD(e, phi) == 1:
        break

n = p * q

flag = b"XXXXXXXXXXXXXXXXXXXXXXX"
pt = bytes_to_long(flag)
ct = pow(pt, e, n)

print(f"n = {n}")
print(f"e = {e}")
print(f"ct = {ct}")

pt = pow(ct, d, n)
decrypted = long_to_bytes(pt)
assert decrypted == flag
