We can see that we are in front of a typical RSA system, in addition they give us the values \(n\), \(e\), \(c\) and \(phi\).

RSA Context.

RSA encrypts a message in decimal format \(m\) we can encrypt it with the following equation.

$$ \begin{equation*} c = m^{e} \text{ (mod } n \text{)} \end{equation*} $$

The public key is formed by \(n\) and \(e\). \(n = p \cdot q\) which are two large prime numbers multiplied (these are stored as the private key).

In order to decrypt the message you need the value \(\phi\)(\(n\)) \(=\) (\(p - 1\)) \( \cdot \) (\(q - 1\)). Then

$$ \begin{equation*} m = c^{d} \text{ (mod } n \text{)} \end{equation*} $$

Vulnerability

In the file provided we are given the value of \(\phi\), using this value we can calculate \(d\) with an alternative expression that encapsulates the relationship between \(\phi(n)\) and \(n\).

To obtain \(d\) let’s define a variable called f as \(f = 2 - \phi + 2\). We are going to explain by formulas why we can recover \(d\).

$$ \begin{equation*} f = 2 \cdot n - \phi + 2 \end{equation*} $$ $$ \begin{equation*} \phi = (p - 1) \cdot (q - 1) \end{equation*} $$

To expand \(\phi\).

$$ \begin{equation*} \phi = pq - (p + q) + 1 \end{equation*} $$

Then.

$$ \begin{equation*} f = 2n - (pq - (p + q) + 1) + 2 \end{equation*} $$ $$ \begin{equation*} f = 2n - pq + p + q - 1 + 2 \end{equation*} $$ $$ \begin{equation*} f = 2n - pq + p + q + 1 \end{equation*} $$

To recover \(d\).

$$ \begin{equation*} d = e^{-1} \text{ (mod } f \text{)} \end{equation*} $$

This means that we can calculate \(d\) using \(f\) instead of \(phi\). The mathematical reason behind this is that \(f\) encapsulates the necessary information about \(n\), \(p\) and \(q\) found in \(phi\).
